FlyByte

FlyByte is a local software application that calculates the optimal flight route among all airports in Mexico using a 52×52 multigraph and a modified Dijkstra algorithm. It also features a fully modeled 3D airplane with seat selection and dynamic fare visualization.

Flutter · Dart · JavaScript · Blender

How It Works

FlyByte processes real airport data provided by the Mexican Federal Civil Aviation Agency (AFAC) to construct a national multigraph. Each airport is represented as a vertex, and each airline forms a distinct layer of connections, enabling FlyByte to calculate the most cost-efficient route across Aeromexico, VivaAerobus, and Volaris.

52 airports mapped across Mexico Real AFAC flight data Modified Dijkstra multigraph algorithm Optimal route selection Cost-based and direct-route search

FlyByte App Flow

A step-by-step visual experience that guides the user from origin/destination selection, through algorithmic fare evaluation, to 3D airplane seat selection and final ticket confirmation.

Route Selection

FlyByte begins with a fully customizable flight search interface where users select origin, destination, seat quantity, and preferred airline criteria. If users choose the “economic route”, the multigraph-based Dijkstra algorithm is executed to compute the cheapest path across all airlines.

  • Origin and destination search
  • Filters by airline and route type
  • Economic vs. direct route modes

Multigraph Visualization

The algorithm uses a 52-vertex multigraph where each airport is connected by edges corresponding to Aeromexico, VivaAerobus, or Volaris. This structure allows FlyByte to compute optimal routes even when no direct flights exist.

  • 52 airports processed
  • Real airline connections
  • Graph-based route exploration

3D Airplane Model

After route evaluation, FlyByte displays a fully modeled 3D airplane rendered in Blender. Users can inspect the aircraft externally before proceeding to seat selection.

  • 3D model created in Blender
  • Perspective and rotation views
  • Real-time preview

3D Cabin View

Users navigate through the interior of the aircraft in a first-person view. Reserved seats appear in red, available ones in gray-blue, and user-selected seats in blue.

  • 3D interior navigation
  • Dynamic seat coloring
  • Interactive seat selection

Seat Selection

Once inside, the user can choose their seats in either 3D mode or traditional layout mode. Each class type (Basic, Regular, Premium) displays its own pricing and availability.

Fare Comparison

FlyByte displays a clear and detailed comparison of ticket prices across multiple times and seating classes, allowing the user to evaluate the best option.

Ticket Confirmation

Once completed, FlyByte generates a clean and fully detailed ticket with the user’s selected route, airline, cost, and aircraft details.

Technical Stack

Developed with Flutter for UI, Dart for logic implementation, JavaScript for algorithmic computation, and Blender for 3D aircraft modeling.

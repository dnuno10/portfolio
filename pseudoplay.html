PseudoPlay

PseudoPlay is a mobile learning tool designed to teach programming logic through an interactive pseudocode interpreter. It supports real-time execution, block-based logic, memory visualization, and line-by-line debugging—all implemented natively in Flutter using a custom Lexer, Parser, Generator, and Execution Engine.

How It Works

PseudoPlay processes pseudocode using a complete custom-built pipeline: a Lexer divides user input into tokens, a Parser verifies syntax and constructs semantic structures, the Generator converts statements into internal tuples, and the Execution Engine simulates program behavior step-by-step with real-time output and memory updates.

PseudoPlay App Flow

From writing pseudocode to executing programs visually, each step of the learning experience is fully interactive and optimized for education.

1. Pseudocode Editor

Users write pseudocode directly in the built-in editor. Syntax is validated on the fly, allowing students to experiment and learn proper structure easily.

2. Block-Based Logic

A visual mode allows younger learners to build logic using predefined blocks, which internally generate valid pseudocode. Conceptually aligned with Scratch but adapted for real pseudocode semantics.

3. Live Execution

Pseudocode is executed line by line using a custom interpreter. Students can visualize outputs, follow control flow, inspect variable updates, and understand conditions and loops through dynamic highlighting.

4. Desk Checking

PseudoPlay includes a desk-checking module, allowing students to manually trace program behavior. This reinforces core programming reasoning skills, such as flow understanding, variable management, and algorithmic thinking.

5. Flashcards

A built-in flashcard system helps students reinforce concepts, memorize common structures, and practice typical exercises found in introductory programming courses.

Technical Architecture

PseudoPlay implements a full pseudocode interpreter built from scratch using a traditional language-processing pipeline. The system processes user input through the following stages:

  • Lexer: Breaks pseudocode into tokens (identifiers, numbers, operators, keywords).
  • Parser: Validates syntactic structure and builds semantic relationships.
  • Generator: Converts statements into internal tuples (Assign, Read, Write, Compare, EndProgram, etc.).
  • Execution Engine: Executes tuples sequentially, updating memory and producing output.
  • Symbol Table: Stores variable names, values, and types throughout execution.
  • Memory Model: Real-time visualization of variables and state changes.
  • Flutter UI: Designed to be intuitive for students learning programming logic.
